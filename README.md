# anonslackbot

Azure-функция для slack-бота анонимной отправки сообщений с помощью слеш-команд

Идея: slack-бот может быть привязан к одному конкретному каналу. У slack-бота есть webhook на https://hooks.slack.com/... 
для отправки сообщений в привязанный канал.  Его можно и нужно использовать напрямую через curl, это режим максимальной анонимности

curl --location --request POST 'https://hooks.slack.com/services/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' \
--header 'Content-Type: application/json' \
--data-raw ' {"text": "Сообщение от бота через webhook"} '

Но для работы слеш-команды типа /03ch  нужен еще один шаг: команда может только посылать все данные куда-то POST-запросом. 

Поэтому azure-функция делает простую вещь: принимает этот POST-запрос, вытаскивает оттуда только text и посылает его на webhook. 
Тогда сообщение попадает в привязанный канал.
